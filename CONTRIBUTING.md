# Contributing to Reachy Mini Skill

First of all, thank you for considering contributing to Reachy Mini SDK Skill! We appreciate your time and effort. This document provides some guidelines for contributing to the project.

## How to Contribute

There are several ways you can contribute to this project:

-   **Reporting Bugs**: If you find a bug, please open an issue and let us know. Make sure to include as much detail as possible, including steps to reproduce the bug.
-   **Suggesting Features**: If you have an idea for a new feature, please open an issue to discuss it. We're always open to new ideas!
-   **Submitting Pull Requests**: If you want to contribute code, you can submit a pull request. Please make sure to follow the guidelines below.

## Reporting Bugs

To report a bug, please open an issue on GitHub. Before you create a new issue, please search the existing issues to see if the bug has already been reported.

When you create a new issue, please include the following information:

-   A clear and descriptive title.
-   A detailed description of the bug, including the steps to reproduce it.
-   The expected behavior and the actual behavior.
-   Information about your environment, such as the operating system, Python version, and the version of the `reachy-mini-sdk-skill` library you're using.

## Suggesting Features

To suggest a new feature, please open an issue on GitHub. Before you create a new issue, please search the existing issues to see if the feature has already been suggested.

When you create a new issue, please include the following information:

-   A clear and descriptive title.
-   A detailed description of the feature you're suggesting.
-   An explanation of why you think the feature would be useful.

## Submitting Pull Requests

If you want to contribute code, you can submit a pull request. Before you submit a pull request, please make sure to:

1.  Fork the repository and create a new branch for your changes.
2.  Install the development dependencies by running `make install`.
3.  Make your changes and make sure the tests pass by running `make validate`.
4.  Format your code by running `make format.`.
5.  Lint your code by running `make lint`.
6.  Submit the pull request.

## Code of Conduct

This project has a Code of Conduct that all contributors are expected to follow. Please make sure to read it before contributing.
